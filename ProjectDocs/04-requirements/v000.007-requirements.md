## Requirements

### R001 — Non-JPEG Format Processing Integration

- The pipeline must support processing of all formats supported at the start of the version cycle and TIFF
- All supported formats must pass through the same pipeline stages
- No format-specific branching is allowed outside metadata extraction
- Processing must produce identical structural outputs across formats

---

### R002 — Metadata Extraction Layer Usage

- Metadata must be extracted exclusively through the metadata extraction layer
- Each supported format must have a corresponding metadata extractor
- Metadata extractors must conform to the metadata_extractors structure
- No metadata extraction logic may exist outside the extraction layer

---

### R003 — EXIF Timestamp Extraction

- EXIF timestamp must be extracted using the following fields in priority order:
  1. DateTimeOriginal
  2. DateTimeDigitized
  3. DateTime
- The first valid field must be used
- Invalid EXIF timestamp fields must be ignored
- Presence of invalid EXIF data must not block fallback to other sources

---

### R004 — EXIF Timezone Extraction

- EXIF timezone must be extracted using the following fields:
  - OffsetTimeOriginal
  - OffsetTimeDigitized
  - OffsetTime
- Timezone must be applied only to the corresponding EXIF timestamp field
- No implicit timezone inference is allowed

---

### R005 — Timestamp Source Precedence

- Timestamp must be resolved using the following strict precedence:
  1. EXIF timestamp (with timezone if available)
  2. Filename-derived timestamp
  3. Folder-derived timestamp
  4. Filesystem timestamp
- The first valid timestamp source must be used
- Lower-precedence sources must not influence resolution

---

### R006 — Timestamp Validity Enforcement

- A timestamp source must be considered valid only if:
  - it can be parsed deterministically
  - it contains year, month, day, hour, minute, and second
  - it satisfies calendar constraints
- Invalid timestamps must be ignored
- Invalid higher-precedence timestamps must not block fallback

---

### R007 — Timestamp Representation Model

- Each valid timestamp source must provide:
  - `naive_timestamp`
  - `timezone_offset` (if available)
  - `timezone_aware_timestamp` (if timezone is available)
  - `utc_timestamp` (if conversion is possible)
- No implicit timezone inference is allowed
- `timezone_aware_timestamp` and `utc_timestamp` must exist only when timezone is available

---

### R008 — Timestamp Comparison Rules

- Timestamp sources are directly comparable only if they share a common representation:
  - UTC with UTC
  - naive with naive
- Timestamps that are not directly comparable must not be compared
- No implicit normalization is allowed for comparison

---

### R009 — Timestamp Inconsistency Detection

- An inconsistency must be reported only if:
  - multiple valid timestamp sources exist
  - the sources are directly comparable
  - their values differ
- No inconsistency diagnostic must be produced for non-comparable timestamps

---

### R010 — Filename Timestamp Extraction

- Filename timestamps must be extracted using the following patterns:

  Full datetime:
  - YYYYMMDD_HHMMSS
  - YYYY-MM-DD_HH-MM-SS
  - YYYY-MM-DD HH-MM-SS

  Date-only:
  - YYYYMMDD
  - YYYY-MM-DD

- Parsing must be left-to-right
- First full match must be used
- Full datetime patterns must have priority over date-only patterns
- Matches must respect character boundaries
- Invalid matches must be rejected
- Date-only timestamps must default to time 00:00:00

---

### R011 — Folder Timestamp Extraction

- Folder timestamps must be extracted from the immediate parent folder only
- Supported patterns must include:

  Single date:
  - YYYYMMDD
  - YYYY-MM-DD
  - YYYY MM DD

  Range:
  - YYYY MM DD - DD
  - YYYY MM DD - MM DD
  - YYYY MM DD - YYYY MM DD
  - YYYY-MM-DD - DD
  - YYYY-MM-DD - MM-DD

- Parsing must be left-to-right
- First full match must be used
- Matches must respect character boundaries
- Invalid matches must be rejected
- Range patterns must use the start date only
- Folder timestamps must default to time 00:00:00

---

### R012 — Filesystem Timestamp Handling

- Filesystem timestamp must be based on file modification time (mtime)
- Filesystem timestamp must be read in UTC
- Filesystem timestamp must be truncated to second precision

---

### R013 — Metadata Diagnostics

- Diagnostics must be generated for each file as applicable
- Diagnostic categories must include:
  - missing timestamp
  - invalid timestamp
  - inconsistent timestamp
- Diagnostics must be deterministic
- Diagnostics must not influence pipeline behavior
- No diagnostic must be produced for non-comparable timestamp sources

---

### R014 — Metadata Normalization Consistency

- Metadata normalization must produce identical data structures across all supported formats
- Normalized metadata must not depend on source format
- All timestamp representations must conform to the defined model

---

### R015 — EXIF Diagnostics Exposure

- The system must expose EXIF-related diagnostics including:
  - missing EXIF data
  - unreadable EXIF fields
  - invalid EXIF fields
- Diagnostics must be deterministic and reproducible

---

### R016 — EXIF Module Structure Alignment

- The EXIF module must be aligned with the metadata_extractors structure
- Alignment must not change existing behavior
- Alignment must not introduce new functionality

---

## Requirement Coverage Rules

- All approved scope elements must be fully covered
- No scope element may be omitted
- No requirement may exist outside approved scope
- No duplication of behavior across requirements is allowed

---

## Validation Criteria

Each requirement must be:

- testable without interpretation
- verifiable against implementation
- unambiguous in all edge cases
- consistent with all other requirements

---

## Constraints

- No modification of existing validated behavior unless explicitly defined
- No introduction of behavior not present in approved scope
- No reliance on implicit logic
- No reliance on environment-dependent behavior
- All behavior must be deterministic
