# Planning Proposal  — v0.5

Starting planning proposal.

---

## Context

- v0.1, v0.2, v0.3 are complete and released (alpha)
- v0.4 introduces workflow standardization (templates, milestones, release discipline)
- All existing behavior (scanner, planner, reporter, CLI) is frozen and must not change
- Scope must come strictly from ProjectDocs/backlog.md
- No changes to existing architecture unless explicitly required

---

## Candidate Scope (from backlog)

- Support additional file formats (PNG, HEIC, RAW, video)
- Improve metadata extraction capabilities
- Introduce contextual grouping based on metadata (camera, location, time proximity)

---

## Goal

Extend PhotoForge beyond JPEG-only processing by introducing deterministic multi-format support, a normalized metadata extraction layer, and deterministic contextual grouping, while preserving the existing pipeline behavior and guarantees.

---

## Responsibilities

- Extend scanning to support additional file formats
- Ensure deterministic handling of each new format
- Introduce a normalized metadata extraction layer across all supported formats
- Introduce deterministic contextual grouping based on extracted metadata
- Preserve existing JPEG behavior exactly
- Maintain deterministic pipeline guarantees

---

## Non-Goals

- No perceptual duplicate detection
- No AI-based classification or grouping
- No changes to canonical selection logic
- No redesign of planner or reporter
- No user-defined rules or configuration system
- No metadata inference or heuristic enrichment

---

## Deliverables

- ``Multi-format scanning support``
- ``Normalized metadata extraction layer``
- ``Format-specific validation and error handling``
- ``Deterministic contextual grouping mechanism``
- ``Deterministic integration into existing pipeline``

---

## Constraints

- No modification of existing validated behavior
- No new architecture beyond minimal, explicit extensions
- No deviation from backlog-defined scope
- Deterministic and minimal changes only

### Pipeline Layering

The pipeline must remain strictly layered:

1. File format support extends scanning only  
2. Metadata extraction is a separate, normalized layer  
3. Duplicate grouping remains strictly hash-based and unchanged  
4. Contextual grouping is a separate deterministic layer  

No step may introduce implicit behavior into another step.

### Metadata Rules

- Metadata extraction is limited to:
  - reading existing metadata
  - normalizing metadata into a consistent structure
- No inference, heuristics, or enrichment allowed
- Missing metadata must not be guessed or approximated

### Grouping Rules

- Contextual grouping must be:
  - deterministic
  - rule-based
  - fully reproducible
- Contextual grouping must not alter:
  - duplicate grouping (hash-based)
  - canonical selection rules

### Pipeline Contract

All supported formats must follow the same pipeline contract:

- timestamp extraction
- hashing (where applicable)
- duplicate grouping (hash-based)
- contextual grouping (separate layer)

---

## Acceptance Criteria

- Scope must come strictly from ProjectDocs/backlog.md
- JPEG behavior remains unchanged and fully deterministic
- New formats are processed deterministically
- Metadata extraction is consistent and reproducible across formats
- Contextual grouping is deterministic and reproducible
- No unintended side effects on existing pipeline
- Duplicate grouping and canonical selection remain unchanged
- Output remains fully deterministic for identical input

---

## Risks

- Format-specific inconsistencies (e.g. EXIF vs non-EXIF metadata)
- Hidden non-determinism in third-party libraries
- Scope creep toward heuristic or implicit behavior
- Implicit coupling between pipeline layers
- Increased implementation complexity across multiple domains

---

## Success Definition

- PhotoForge supports multiple file formats deterministically
- Metadata is extracted and normalized consistently across formats
- Contextual grouping operates as a separate deterministic layer
- Existing pipeline remains unchanged for JPEG processing
- No implicit behavior or hidden coupling is introduced
- The system remains predictable, reproducible, and minimal
