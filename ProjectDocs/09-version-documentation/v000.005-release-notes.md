# Release Notes — v000.005

Version: v0.5.0  
Date: 2026-04-17  
Status: stable / accepted

---

## Overview

v0.5.0 extends PhotoForge from a JPEG-only deterministic duplicate-planning tool into a broader, still strictly deterministic system with:

- recognized multi-format scanning
- metadata abstraction and normalization
- contextual grouping as an independent structural output
- explicit pipeline orchestration
- additive reporter and CLI support for contextual output

This release preserves the existing deterministic planning guarantees while formalizing architecture, contracts, and documentation across the full system.

---

## Highlights

### Multi-format recognition

The scanner now recognizes additional non-JPEG formats:

- PNG
- HEIC / HEIF
- RAW formats (`CR2`, `NEF`, `ARW`)
- Video formats (`MP4`, `MOV`)

These formats are recognized deterministically and classified explicitly, while active processing remains limited to JPEG files.

### Metadata normalization

Metadata handling now passes through a normalized abstraction layer with explicit validation rules.

Key properties:

- normalized structure contains only:
  - `timestamp`
  - `timestamp_source`
- timestamps must be naive
- no fallback, transformation, inference, or mutation is introduced by normalization

### Contextual grouping

v0.5.0 introduces contextual grouping as a separate structural output:

- computed from valid `FileRecord` objects
- deterministic and reproducible
- independent from duplicate grouping
- independent from canonical selection and planning
- exposed only when explicitly requested

### Explicit pipeline orchestration

The runtime now includes a dedicated orchestration layer:

- scanner
- contextual grouping
- planner

This orchestration is performed by `run_pipeline(...)`, which returns:

- `PlanResult`
- `ContextualGrouping`

as separate outputs.

### Contextual reporter output

Reporter and CLI now support optional contextual output via:

`--context`

This output is additive only and does not alter planning behavior or default output semantics.

## Functional Changes

### Scanner

- extended deterministic extension recognition
- explicit distinction between:
  - processable JPEG files
  - recognized but non-processable formats
  - unsupported formats
- corrupt file classification remains deterministic and explicit

### Metadata

- introduced `NormalizedMetadata`
- added explicit normalization contract
- preserved existing JPEG timestamp extraction behavior
- enforced strict naive timestamp validation

### Grouping

- introduced contextual grouping structures and rules
- defined deterministic ordering and grouping behavior
- grouped records using explicit temporal adjacency logic
- ensured grouping remains independent from duplicate handling

### Pipeline

- introduced dedicated orchestration module
- formalized current double-scan runtime architecture
- preserved planner behavior while adding parallel grouping output

### Reporter

- extended console and JSON reporting to optionally include contextual grouping
- preserved default output behavior when contextual output is not requested
- ensured deterministic JSON serialization

### CLI

- added `--context`
- explicitly propagates contextual grouping to reporter
- continues to derive `CorruptFile` from scanner skip reasons before pipeline execution

## Determinism Guarantees

v0.5.0 preserves and strengthens deterministic behavior across the system.

For identical input, the system guarantees identical:

- scan classification
- valid `FileRecord` construction
- duplicate grouping
- canonical selection
- planning results
- contextual grouping
- console output
- JSON output

No randomness, implicit ordering, or environment-dependent behavior is introduced.

## Compatibility Notes

- Existing deterministic JPEG workflow remains intact
- Core planning semantics from earlier versions are preserved
- Contextual grouping is additive and optional

## Constraints

v0.5.0 still intentionally does **not** include:

- full non-JPEG processing
- perceptual duplicate detection
- file deletion
- metadata rewriting
- overwrite behavior
- concurrency
- external integrations
