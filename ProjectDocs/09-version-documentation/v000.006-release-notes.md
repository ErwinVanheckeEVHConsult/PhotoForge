# Release Notes — v000.006

## Version

- Human-readable version: `v0.6`
- Document version: `v000.006`

---

## Summary

v0.6 completes the workflow-alignment release for PhotoForge.

This release does not introduce planned runtime feature changes. Its purpose is to make the project workflow, documentation structure, templates, enforcement rules, and repository organization deterministic, complete, and internally consistent.

Primary outcomes:

- deterministic versioning and commit naming rules defined
- workflow steps and transitions made explicit
- documentation storage aligned with workflow structure
- required templates completed, aligned, and enforced
- workflow enforcement rules made explicit and blocking
- system documentation domain formally defined
- system documentation migrated into its authoritative location
- documentation corrected to match implementation and actual repository state
- backlog cleaned to remove work completed by v0.6

---

## Scope

v0.6 is limited to workflow and documentation alignment.

Included scope:

- workflow definition and enforcement
- template coverage and template enforcement
- repository documentation structure alignment
- system documentation classification and migration
- documentation consistency corrections
- backlog consistency cleanup

Excluded scope:

- no new runtime features
- no new processing formats
- no behavioral redesign of scanner, planner, pipeline, reporter, or CLI
- no release engineering automation beyond documentation and workflow definition

## Key Repository Changes

### Workflow

`development-workflow.md` now defines:

- deterministic version assignment and propagation
- explicit workflow transitions
- document storage rules
- template usage and template enforcement
- workflow enforcement and blocking conditions
- system documentation domain and storage rules
- milestone start and completion definitions

### Templates

Workflow templates now provide complete and aligned structure for:

- documentation milestones
- implementation milestones
- milestone checklists
- milestones overview

### Documentation Structure

Project documentation is now organized into explicit domains, including:

- `01-backlog`
- `02-version-scope`
- `03-scope-approval`
- `04-requirements`
- `05-milestones`
- `09-version-documentation`
- `00-system-documentation`

### System Documentation

System-level reference documentation is now stored under:

- `ProjectDocs/00-system-documentation/`

This includes technical and behavioral reference documents such as scanner, planner, pipeline, reporter, model, metadata, CLI, validation, and related extractor documentation.

- none intended

## Determinism and Validation Improvements

This release strengthens determinism by making explicit:

- workflow ordering
- milestone ordering
- template selection by milestone type
- blocking validation conditions
- documentation storage rules

It also removes repository ambiguity by ensuring that documentation artifacts have a single authoritative location and that milestone identity, overview, checklist, and implementation history remain consistent.
