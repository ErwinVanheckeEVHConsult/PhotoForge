# Milestone 20 — contributor-workflow

Starting milestone 20.

---

## Context

- v0.1, v0.2, and v0.3 are complete and released (alpha)
- All v0.1–v0.3 behavior is frozen and must not change
- v0.4 introduces workflow standardization only (no runtime changes)
- No changes are allowed to scanner, planner, reporter, operations, or CLI
- Backlog item: Documentation for contributors :contentReference[oaicite:0]{index=0}
- Planning proposal for v0.4 is defined and approved
- Milestone 18 introduced planning and milestone templates
- Milestone 19 introduced validation, release, and changelog templates
- Scope is strictly limited to documenting the development workflow

---

## Goal

Define a deterministic and reproducible development workflow that explains how to plan, implement, validate, and release PhotoForge versions using the defined templates.

---

## Responsibilities

- Define the complete development lifecycle:
  - backlog management
  - planning
  - milestone definition
  - implementation
  - validation
  - commit
  - release
- Define document storage locations for all artifacts
- Define deterministic naming conventions for all documents
- Define template usage rules and constraints
- Ensure workflow:
  - is strictly procedural
  - contains no ambiguity
  - requires no implicit knowledge

---

## Deliverables

- `ProjectDocs/development-workflow.md`

The document must:

- define lifecycle steps in strict sequence
- define document locations and naming conventions
- define validation and release process
- align with all templates introduced in MS18 and MS19
