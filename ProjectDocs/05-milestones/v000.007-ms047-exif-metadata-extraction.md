# Milestone - v000.007-ms047 - exif-metadata-extraction

## Type

implementation

---

## Context

- Metadata extraction layer structure is defined in MS046
- EXIF extraction currently exists in:

  ``src/photoforge/exif.py``

- The current pipeline:

  - directly calls ``extract_timestamp(...)`` from ``exif.py``
  - applies normalization via ``normalize_metadata(...)``

- Non-JPEG extractors already conform to a unified interface under:

  ``src/photoforge/metadata_extractors/``

- EXIF extraction is not aligned with the metadata extraction layer

---

## Goal

Align EXIF metadata extraction with the metadata extraction layer.

The outcome must ensure:

- EXIF extraction conforms to the extractor interface
- EXIF extraction is integrated into ``metadata_extractors``
- EXIF timestamp extraction behavior remains unchanged
- Pipeline uses the EXIF extractor via the metadata extraction layer
- No regression is introduced in existing behavior

---

## Affected Components

- ``src/photoforge/exif.py``
- ``src/photoforge/metadata_extractors/`` (new EXIF extractor module)
- ``src/photoforge/scanner.py``

---

## Inputs

- ``path: Path``
- ``mtime_timestamp: float``

---

## Outputs

- ``tuple[datetime, str]``

Where:

- ``datetime`` is a naive timestamp
- ``str`` is the timestamp source identifier

---

## Processing Rules

### Extractor Integration

- EXIF extraction must be exposed through a new extractor under:

  ``metadata_extractors``

- The extractor must implement:

  ```python
  (path: Path, mtime_timestamp: float) -> tuple[datetime, str]
  ```

- The extractor must reuse existing EXIF logic without modification

---

### Timestamp Extraction

EXIF timestamp extraction must follow the existing deterministic precedence:

1. ``exif_datetimeoriginal`` (tag 36867)
2. ``exif_datetimedigitized`` (tag 36868)
3. ``exif_datetime`` (tag 306)

Behavior:

- The first valid timestamp must be returned
- A value is valid only if it can be parsed using:

  ```python
  "%Y:%m:%d %H:%M:%S"
  ```

- Invalid values must be ignored

---

### Fallback Behavior

If no valid EXIF timestamp is found:

```python
datetime.fromtimestamp(mtime_timestamp), "mtime"
```

Rules:

- fallback must always succeed
- fallback must be deterministic
- EXIF errors must not block fallback

---

### Timezone Handling

- No timezone extraction is implemented

- EXIF timestamps must remain naive

- No timezone parsing, inference, or transformation is allowed

---

### Pipeline Alignment

- ``scanner.py`` must invoke EXIF extraction via the metadata_extractors structure

- Direct import:

  ```python
  from .exif import extract_timestamp
  ```

  must be removed

- Scanner must call the EXIF extractor instead

---

## Invariants

- Output timestamp must remain:
  - naive datetime
  - deterministic

- Existing extraction behavior must remain unchanged

- No implicit timezone inference may be introduced

- Non-JPEG extractors must remain unchanged
