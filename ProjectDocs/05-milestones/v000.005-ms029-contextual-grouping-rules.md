# Milestone — v000.005-ms029 — contextual-grouping-rules

Starting milestone.

---

## Context

- v0.1–v0.3 define a deterministic pipeline for duplicate detection and canonical selection
- v0.5 introduces contextual grouping as a separate structural layer
- MS028 defines the contextual grouping model:
  - ContextualGroup
  - ContextualGrouping
  - record_ref
  - group_id contract
  - structural invariants (partition, ordering, identity)
- No grouping rules or execution logic exist yet
- The model guarantees:
  - immutability
  - determinism
  - separation from pipeline behavior
- This milestone defines the deterministic rules used to construct contextual groups

---

## Goal

Define a complete, deterministic, and explicit set of contextual grouping rules that construct valid ContextualGrouping structures from a set of FileRecord objects, without introducing any pipeline integration.

---

## Responsibilities

- Define deterministic input ordering for grouping
- Define grouping adjacency rule
- Define grouping boundary rule
- Define fixed grouping constants
- Ensure grouping rules produce valid MS028-compliant structures
- Ensure grouping is fully deterministic and reproducible
- Ensure no interaction with duplicate grouping or planner logic

---

## Deliverables

- Deterministic input ordering definition
- Explicit grouping adjacency rule
- Explicit group boundary rule
- Definition of grouping constant TIME_WINDOW_SECONDS
- Formal grouping construction algorithm
- Validation that output conforms to MS028 model

---

## Behavior

### Required behavior

- Contextual grouping must be constructed from valid FileRecord objects only
- Corrupt files must not participate in grouping
- Grouping must be:
  - deterministic
  - explicit
  - fixed
- No heuristic, probabilistic, or dynamic behavior is allowed
- No dependency on scan order or filesystem traversal is allowed

---

### Input domain

Let R be the set of valid FileRecord objects.

Each FileRecord contains:

- path
- timestamp
- timestamp_source

All timestamps are naive and comparable.

---

### Deterministic ordering

Before grouping, define ordered sequence S:

1. Sort by timestamp ascending
2. If equal, sort by str(path) lexicographically ascending

Rules:

- Ordering must be total and deterministic
- No alternative ordering is allowed

---

### Grouping constant

Define:

TIME_WINDOW_SECONDS = 300

Rules:

- Must be defined as an integer number of seconds
- Must be constant
- Must not depend on dataset, runtime, or environment

---

### Adjacency rule

For consecutive records si and s(i+1) in S:

adjacent(si, s(i+1)) = true
if (s(i+1).timestamp - si.timestamp).total_seconds() <= TIME_WINDOW_SECONDS

Otherwise:

adjacent(si, s(i+1)) = false

Because S is sorted by timestamp ascending, (s(i+1).timestamp - si.timestamp) is always >= 0.

Rules:

- Only forward difference is allowed
- No use of absolute value
- Comparison must use datetime arithmetic

---

### Group boundary rule

A new group starts:

- at s1
- at any s(i+1) where adjacent(si, s(i+1)) = false

A group continues:

- from si to s(i+1) when adjacent(si, s(i+1)) = true

---

### Group construction

Groups are defined as maximal contiguous subsequences of S under the adjacency rule.

Each group:

- contains one or more consecutive elements of S
- is maximal (cannot be extended)

---

### Mapping to MS028 model

For each group:

1. Convert each FileRecord to record_ref:
record_ref = str(file_record.path)
2. Derive member references from the group and sort them lexicographically for MS028/model compliance:
member_refs = tuple(sorted((r_a, r_{a+1}, ..., r_b)))
3. Construct ContextualGroup:
group_id = compute_group_id(member_refs)
4. Collect all groups into:

ContextualGrouping(groups=...)

---

### Structural guarantees

If R is empty, the output must be:

ContextualGrouping(groups=())

The constructed grouping must satisfy:

- Coverage:
  - all records in R are included in the output grouping
- Exclusivity:
  - each record appears in exactly one group
- No omission:
  - no record in R may be omitted
- Non-emptiness:
  - no group is empty
- Determinism:
  - identical input produces identical grouping
- Ordering:
  - member_refs is sorted lexicographically for MS028/model compliance
- Identifier:
  - group_id matches MS028 contract
- Container ordering:
  - groups sorted lexicographically by group_id

---

### Rules

- Deterministic behavior only
- No implicit grouping
- No dynamic thresholds
- No inference or enrichment
- No dependency on external state
- No modification of FileRecord
- No interaction with duplicate grouping

---

## Constraints

- No changes to scanner behavior
- No changes to hashing logic
- No changes to duplicate grouping
- No changes to canonical selection
- No changes to planner behavior
- No changes to reporter behavior
- No modification of FileRecord
- No integration into active pipeline
- No metadata beyond:
  - timestamp
  - timestamp_source

---

## Acceptance Criteria

- Input ordering is explicitly defined and deterministic
- Grouping rule is explicit and fixed
- Boundary rule is explicit and fixed
- TIME_WINDOW_SECONDS is defined as constant integer
- Adjacency rule uses forward timestamp difference only
- No use of absolute difference
- Group construction produces maximal contiguous subsequences
- Output strictly conforms to MS028 model
- Each record appears in exactly one group
- No group is empty
- Group identifiers are deterministic and correct
- Output is identical for identical input
- No unintended side effects

---

## Risks

- Incorrect ordering leading to inconsistent grouping
- Threshold misdefinition leading to unstable grouping
- Accidental use of non-deterministic logic
- Violation of MS028 structural constraints

---

## Success Definition

- Grouping rules are fully defined, explicit, and deterministic
- Group construction is unambiguous and reproducible
- Output structures fully comply with MS028
- The system is ready for grouping execution and integration in the next milestone
