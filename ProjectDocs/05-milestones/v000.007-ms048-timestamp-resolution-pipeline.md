# Milestone - v000.007-ms048 - timestamp-resolution-pipeline

## Type

implementation

---

## Context

- MS046 defines the metadata extraction layer structure
- MS047 aligns JPEG EXIF extraction with ``metadata_extractors``
- Extractors exist for:
  - JPEG
  - HEIC
  - PNG
  - RAW
  - video

- The current pipeline:

  - processes only JPEG files
  - skips other supported formats
  - does not use a unified extraction flow

- Timestamp extraction is currently single-source per format:
  - EXIF for JPEG
  - mtime fallback for all formats

- There is no unified multi-format timestamp resolution pipeline

- ``metadata.py`` remains the normalization boundary and is not expanded in this milestone

---

## Goal

Implement a unified timestamp resolution pipeline for all supported formats.

The outcome must ensure:

- all supported formats are processed through the same extraction pipeline
- extraction is routed exclusively through ``metadata_extractors``
- timestamp source precedence is explicitly defined and enforced
- timestamp validity is enforced before selection
- deterministic selection of the first valid timestamp source
- fallback behavior is consistent across all formats

---

## Affected Components

- ``src/photoforge/scanner.py``
- ``src/photoforge/metadata_extractors/``

---

## Inputs

- ``path: Path``
- ``mtime_timestamp: float``

---

## Outputs

- ``tuple[datetime, str]``

Where:

- ``datetime`` is a naive timestamp
- ``str`` is the timestamp source identifier

---

## Processing Rules

### Supported Format Handling

Supported formats:

- ``.jpg``, ``.jpeg``
- ``.png``
- ``.heic``, ``.heif``
- ``.cr2``, ``.nef``, ``.arw``
- ``.mp4``, ``.mov``

Rules:

- All supported formats must be processed via their corresponding extractor
- No supported format may be skipped
- The concept of "recognized but not processable" must be removed

---

### Unified Extraction Routing

- Scanner must route all supported formats through ``metadata_extractors``

- Extractor selection must be based on file extension

- No format-specific extraction logic may exist outside ``metadata_extractors``

---

### Timestamp Resolution

- Extractors are responsible for full timestamp resolution

- Each extractor must return:
  - a valid timestamp from its format-specific source
  - or fallback to ``mtime`` if no valid timestamp is available

- The pipeline must not implement additional fallback or precedence logic

Rules:

- the first valid timestamp must be selected
- precedence must be consistent across all formats

---

### Timestamp Validity Enforcement

A timestamp is valid if:

- it is a ``datetime`` instance
- it is naive (no timezone)
- it conforms to extractor constraints

Invalid timestamps must be ignored

---

### Fallback Behavior

If no valid extractor timestamp exists:

```python
datetime.fromtimestamp(mtime_timestamp), "mtime"
```

Rules:

- fallback must be deterministic
- fallback must always succeed
- fallback behavior must be identical across all formats

---

### Pipeline Consistency

The pipeline must:

- apply identical processing steps for all formats
- follow:

``
extractor → normalize_metadata → FileRecord
``

The pipeline must not:

- introduce format-specific branching after extraction
- alter normalization behavior
- introduce implicit fallback logic

---

## Invariants

- Output timestamp must remain:
  - naive
  - deterministic

- Existing EXIF behavior must remain unchanged

- Non-JPEG extractor behavior must remain unchanged

- No implicit timezone inference may be introduced
