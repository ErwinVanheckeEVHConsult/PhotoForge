# Milestone - v000.007-ms055 - exif-diagnostics-exposure

## Type

implementation

---

## Context

- Extraction layer (MS047–MS053) produces timestamp candidates from multiple sources
- Some extraction failures do not produce candidates (e.g. missing or unreadable EXIF data)
- Timestamp resolution (MS048) operates only on valid candidates
- Timestamp comparison (MS050) produces diagnostics from candidate comparison
- Diagnostics currently do not expose EXIF-specific extraction failures

R015 requires explicit exposure of EXIF-related diagnostics:

- missing EXIF data
- unreadable EXIF fields
- invalid EXIF fields

Affected components:

- src/photoforge/metadata_extractors/exif.py
- src/photoforge/timestamp_diagnostics.py
- src/photoforge/scanner.py
- src/photoforge/model.py (read-only extension if needed)

---

## Goal

Expose deterministic and reproducible EXIF-related diagnostics, including extraction-level failure states, without affecting extraction, resolution, or normalization behavior.

---

## Affected Components

- `metadata_extractors/exif.py` (diagnostic emission)
- `timestamp_diagnostics.py` (aggregation / structure)
- `scanner.py` (integration only)

Untouched:

- `metadata.py`
- `timestamp_resolution.py`
- other extractors

---

## Behavior Specification

### Inputs

- EXIF extraction process (during candidate extraction)
- `valid_candidates: tuple[TimestampCandidate, ...]`

---

## Processing Rules

- EXIF diagnostics must be produced during extraction
- Diagnostics must be produced even if no candidate is created
- Diagnostics must not influence:
  - candidate creation
  - candidate validation
  - resolution outcome
  - normalization
- Diagnostics must be deterministic and reproducible

---

## EXIF Diagnostic Types

The system must produce the following EXIF diagnostics:

### 1. Missing EXIF Data

Condition:

- No EXIF metadata is present in the file

Diagnostic:

- `"exif_missing"`

---

### 2. Unreadable EXIF Data

Condition:

- EXIF metadata exists but cannot be parsed or accessed

Diagnostic:

- `"exif_unreadable"`

---

### 3. Invalid EXIF Fields

Condition:

- EXIF fields exist but:
  - contain invalid format
  - cannot be parsed into a valid timestamp
  - fail validation rules

Diagnostic:

- `"exif_invalid"`

### Diagnostic Classification Rules

- "exif_unreadable":
  - EXIF structure cannot be accessed or decoded at all

- "exif_invalid":
  - EXIF field is readable but contains invalid or unparsable timestamp data

---

## Diagnostic Representation

EXIF diagnostics must be represented as structured entries.

Minimum structure:

- `source_kind = "exif"`
- `diagnostic_type ∈ {missing, unreadable, invalid}`
- optional `field_name` (if applicable)

The structure must be deterministic and must not depend on runtime conditions beyond input data.

---

## Candidate-Based Diagnostics

In addition to extraction diagnostics:

- Candidate comparison diagnostics must still be produced (MS050)
- EXIF candidates participate in comparisons like any other source
- No EXIF-specific comparison logic is allowed

---

## Integration

- EXIF diagnostics must be included in the final diagnostics output
- Scanner must aggregate:
  - extraction diagnostics
  - comparison diagnostics

---

## Edge Cases

- File without EXIF → must produce `"exif_missing"`
- Corrupt EXIF → must produce `"exif_unreadable"`
- Invalid EXIF timestamp → must produce `"exif_invalid"`
- Valid EXIF timestamp → no extraction diagnostic
- Multiple EXIF issues → multiple diagnostics allowed

---

## Invariants

- No modification of candidate data
- No influence on resolution outcome
- No implicit fallback behavior
- Diagnostics must be deterministic for identical inputs

---

## Responsibilities

- Emit EXIF extraction diagnostics
- Aggregate diagnostics into final output
- Ensure deterministic behavior
- Preserve separation between:
  - extraction diagnostics
  - comparison diagnostics

---

## Deliverables

- EXIF diagnostic emission in extractor
- Diagnostics aggregation in pipeline
- Deterministic output structure

---

## Constraints

- No changes outside defined scope
- No modification of existing pipeline logic
- No duplication of resolution logic
- No implicit or inferred behavior

---

## Validation Criteria

- Missing EXIF produces `"exif_missing"`
- Unreadable EXIF produces `"exif_unreadable"`
- Invalid EXIF produces `"exif_invalid"`
- Diagnostics are deterministic across runs
- Diagnostics do not affect timestamp selection
- Comparison diagnostics still function correctly

- Timestamp resolution (MS048)
