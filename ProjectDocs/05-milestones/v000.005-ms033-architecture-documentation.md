### Interaction Rules

When performing documentation validation or creation:

- If required information is missing, unclear, or cannot be verified from available context,
  the assistant must explicitly request additional inputs from the user.

- The assistant must ask for concrete artifacts, including but not limited to:
  - `.md` files (documentation)
  - `.py` files (implementation)
  - CLI command outputs
  - test outputs (e.g. E2E golden files)
  - any other relevant project files

- The assistant must not:
  - guess behavior
  - infer undocumented logic
  - proceed with partial or assumed information

- Requests must be specific and actionable:
  - identify exactly which file or output is needed
  - explain why it is required for validation or documentation

- Documentation work must pause until sufficient information is provided

## Architecture Document Requirements

`architecture.md` must:

### Pipeline Flow

Describe the execution flow using exact implemented interfaces:

```text
CLI
  → run_pipeline(input_path, ...)
      → scan_directory(...) → ScanResult
      → plan_files(scan_result.records, ...) → PlanResult
      → compute_contextual_grouping(scan_result.records) → ContextualGrouping
      → return:
        - PlanResult
        - ContextualGrouping
  → reporter(plan_result, contextual_grouping, ...)
```

Must explicitly state:

- contextual grouping is computed from the `FileRecord` set
- contextual grouping does not influence planner behavior
- contextual grouping is passed separately to reporter

---

### Model Layer

- Must document all dataclasses in `model.py`
- Must describe:
  - invariants
  - immutability
  - determinism guarantees

---

### Determinism

Must explicitly define:

- identical input → identical output
- all ordering is explicitly defined
- no reliance on implicit ordering
- no environment-dependent behavior
- no randomness

---

### Component Boundaries

Must clearly define responsibilities and prohibitions for:

- scanner
- planner
- grouping
- reporter
- pipeline orchestration
