# Milestone — v000.006-ms035 — commit-naming-convention

## Type

documentation

---

## Context

- Commit naming is not standardized
- Commit messages may vary across milestones and releases

---

## Goal

Define a deterministic commit naming convention for milestone and version commits.

---

## Affected Documents

- ProjectDocs/development-workflow.md

---

## Definitions

- Milestone commit format:
  - `MS<nnn>: <short-title>`

- Version commit format:
  - `v<mmm>.<nnn>: <description>`

- Commit constraints:
  - Must include version or milestone identifier
  - Must not contain free-form structure

---

## Invariants

- Existing validated rules must remain unchanged unless explicitly redefined
- No implicit commit formats allowed
- All changes must preserve consistency with:
  - development-workflow.md
  - requirements
  - milestones overview

---

## Responsibilities

- Define commit naming rules
- Ensure deterministic commit structure

---

## Deliverables

- ProjectDocs/development-workflow.md updated with “Commit Naming Convention” section
- Explicit definition of:
  - milestone commit format
  - version commit format

---

## Constraints

- No runtime changes
- No tooling assumptions

---

## Validation Criteria

- Commit naming rules exist
- Milestone commit format is defined
- Version commit format is defined
- No ambiguity in commit structure

---

## Non-Goals
