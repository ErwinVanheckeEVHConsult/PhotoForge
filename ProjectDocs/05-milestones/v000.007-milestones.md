### ``MS048 - timestamp-resolution-pipeline``

**Type:**

- implementation

**Summary:**

- Introduce a unified timestamp resolution pipeline across all supported formats
- Define and implement a **timestamp candidate model**:
  - Each source (EXIF, filename, folder, filesystem) produces a candidate
  - Each candidate contains:
    - naive timestamp
    - timezone offset (if available)
    - source identifier
- Ensure extractors can produce **multiple timestamp candidates**
- Ensure all candidates are preserved for:
  - precedence evaluation
  - comparison
  - diagnostics
- Implement timestamp source precedence and validity enforcement:
  - Only valid candidates participate in resolution
  - The first valid candidate by precedence becomes the **primary timestamp**
- Prevent format-specific branching outside metadata extraction
- Ensure deterministic behavior and ordering across all sources
- Ensure pipeline output contains:
  - resolved primary timestamp
  - complete set of valid timestamp candidates

**Covered Requirements:**

- R001 — Non-JPEG Format Processing Integration
- R005 — Timestamp Source Precedence
- R006 — Timestamp Validity Enforcement
