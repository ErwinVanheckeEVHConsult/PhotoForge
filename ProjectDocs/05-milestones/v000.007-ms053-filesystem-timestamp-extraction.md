## Behavior Specification

### Inputs

- `<file_path>` as string (full path to file)

---

### Processing Rules

- Extraction must operate on filesystem metadata only
- No parsing of filename or folder name
- Extraction must be deterministic
- No transformation or normalization of timestamp values
- No interpretation of timestamp semantics
- All available filesystem timestamps must be considered independently
- Missing timestamps must be ignored (not produced)

---

### Extraction Mechanism

Filesystem timestamps must be obtained using Python's standard library:

- ``os.stat(file_path)``

The following attributes must be used:

- ``st_mtime`` → modification time
- ``st_ctime`` → change time (platform-dependent semantics)
- ``st_birthtime`` → creation time (if available)

Conversion rules:

- timestamps must be converted using:
  - ``datetime.fromtimestamp(...)``
- resulting datetime must be treated as naive
- no timezone conversion must be applied

Availability rules:

- if ``st_birthtime`` is not available → do not produce candidate
- all other attributes must be used if present

---

### Supported Sources (as defined in R012)

- modification time (mtime)
- change time (ctime)
- creation/birth time (birthtime, if available)

---

### Outputs

- ``tuple[TimestampCandidate, ...]``

Zero to multiple candidates may be returned.

Each candidate must contain:

- ``source_kind = "filesystem"``
- ``source_detail``:
  - "filesystem_mtime"
  - "filesystem_ctime"
  - "filesystem_birthtime"
- ``naive_timestamp`` = datetime derived from filesystem value
- ``precision = "datetime"``
- ``timezone_offset = None``

---

### Source Detail Mapping

- mtime → filesystem_mtime  
- ctime → filesystem_ctime  
- birthtime → filesystem_birthtime  

---

### Edge Cases

- Missing timestamp → candidate not produced
- Platform differences must not be normalized:
  - ctime semantics differ across OS
- Invalid timestamp values → must be rejected
- No available timestamps → empty result
- Multiple timestamps → all valid candidates must be returned
