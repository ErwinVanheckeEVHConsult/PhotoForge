## Validation

- Input domain is explicitly defined as valid FileRecord objects only
- Corrupt files are explicitly excluded from grouping
- Input ordering is defined exactly as:
  - timestamp ascending
  - if equal: str(path) lexicographically ascending
- Ordering is total and deterministic
- No alternative ordering rules are defined

- TIME_WINDOW_SECONDS is defined as a fixed integer constant
- TIME_WINDOW_SECONDS does not depend on dataset, runtime, or environment

- Adjacency is defined only for consecutive elements in the ordered sequence
- Adjacency rule is defined using forward timestamp difference only:
  - (s(i+1).timestamp - si.timestamp) <= TIME_WINDOW_SECONDS
- No use of absolute difference (abs) is defined
- Timestamp comparison uses datetime arithmetic
- All timestamps are naive and mutually comparable

- Group boundary rule is explicitly defined
- Group construction is defined as maximal contiguous subsequences
- Each group contains one or more consecutive elements
- Groups are maximal and cannot be extended

- Each FileRecord is assigned to exactly one group
- No FileRecord appears in multiple groups
- No group is empty

- record_ref is defined as str(file_record.path)
- member_refs is sorted lexicographically for MS028/model compliance
- member_refs contains only valid record_ref values
- member_refs contains unique values within a group

- group_id is computed using the MS028 canonical contract
- group_id is derived only from member_refs
- group_id is deterministic and stable

- ContextualGroup objects are constructed correctly
- ContextualGrouping.groups contains all groups
- groups are sorted lexicographically by group_id

- Output structure fully conforms to MS028 model invariants
- No grouping depends on scan order or filesystem traversal
- No external state or environment influences grouping

- No heuristic, probabilistic, or dynamic behavior is introduced
- No metadata beyond timestamp and timestamp_source is used
- No implicit rules or undocumented assumptions are introduced

- Deterministic behavior is preserved
- Identical input produces identical grouping output

- No unintended side effects
