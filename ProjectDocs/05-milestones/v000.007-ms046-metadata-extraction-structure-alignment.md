# Milestone - v000.007-ms046 - metadata-extraction-structure-alignment

## Type

documentation

---

## Context

- Metadata extraction logic currently exists in multiple locations:
  - ``src/photoforge/exif.py`` (JPEG EXIF extraction)
  - ``src/photoforge/metadata_extractors/`` (format-specific extractors for HEIC, PNG, RAW, video)
  - ``src/photoforge/scanner.py`` directly imports ``.exif``
  - ``src/photoforge/metadata.py`` wraps EXIF extraction but is not part of the active pipeline

- The metadata extraction layer exists but is not fully defined as a formal contract
- JPEG/EXIF extraction is not integrated into the metadata extraction layer

---

## Goal

Define a complete, explicit, and verifiable metadata extraction layer structure.

The outcome must ensure:

- a single, unified metadata extraction layer definition
- all format-specific extraction logic is formally assigned to that layer
- the structure and contracts are fully defined in system documentation
- no ambiguity remains regarding extractor location, interface, or responsibilities

---

## Affected Documents

- ``ProjectDocs/00-system-documentation/metadata-extraction-layer.md`` (created)

---

## Definitions

### Metadata Extraction Layer

- The metadata extraction layer defines the future exclusive entry point for all metadata extraction logic
- All metadata extraction must be performed through this layer once implemented
- No module outside this layer may directly access format-specific metadata once implemented

---

### Extractor Location

- All format-specific metadata extractors must reside under:

  ``src/photoforge/metadata_extractors/``

- Each supported format must have a dedicated extractor module

---

### Extractor Interface

Each extractor must:

- be a pure function
- accept:
  - ``path: Path``
  - ``mtime_timestamp: float``
- return:

  ``tuple[datetime, str]``

Where:

- ``datetime`` is a naive timestamp
- ``str`` is the timestamp source identifier

---

### Extractor Behavior Constraints

- Extractors must:
  - be deterministic
  - not perform normalization
  - not perform cross-source fallback beyond their defined scope
  - not depend on environment-specific behavior

- Extractors must not:
  - modify filesystem state
  - depend on global state
  - introduce implicit logic

---

### Layer Boundary

- The metadata extraction layer must:

  - select the appropriate extractor based on file format
  - invoke the extractor
  - return raw extracted metadata

- The layer must not:
  - perform normalization
  - perform comparison
  - perform diagnostics

---

### EXIF Alignment Rule

- JPEG EXIF extraction must be implemented as a format-specific extractor within:

  ``metadata_extractors``

- Standalone EXIF modules outside this layer must not be used by the pipeline once alignment is implemented

---

## Invariants

- Existing validated behavior must remain unchanged unless explicitly redefined
- Output structure must remain ``tuple[datetime, str]``
- Timestamp must remain naive
- No implicit timezone inference may be introduced
- No format-specific branching may occur outside the metadata extraction layer once implemented
