## Validation

- Scanner includes files with supported extensions:
  - .jpg, .jpeg (existing)
  - .png
  - .heic, .heif
  - .cr2, .nef, .arw
  - .mp4, .mov
- Extension matching is case-insensitive and based on normalized lowercase suffix
- Only supported extensions are included; all others are ignored
- No file content inspection is introduced for format detection (extension-based only)
- JPEG processing behavior remains unchanged
- Non-JPEG formats pass through the pipeline without format-specific handling
- Timestamp extraction behavior remains unchanged:
  - Non-JPEG formats fall back to mtime where EXIF is unavailable
- No changes are introduced to hashing behavior in this milestone
- Files are only classified as corrupt on deterministic processing failure (e.g. I/O, hashing failure)
- Hashing is applied identically to all supported file formats
- Newly supported formats are not classified as corrupt due to lack of format-specific support
- No unintended file types are included due to incorrect extension handling
- No regression in existing scanner behavior
