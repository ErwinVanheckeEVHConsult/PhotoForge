# Milestone — v000.006-ms040 — template-enforcement

## Type

documentation

---

## Context

- Required templates now exist
- Template structure has been aligned with workflow requirements
- Workflow-generated documents may still deviate from their required templates
- Placeholder leakage and manual structure deviation must be prevented explicitly

---

## Goal

Define and enforce mandatory template usage for all workflow-generated documents.

---

## Affected Documents

- ProjectDocs/development-workflow.md
- ProjectDocs/templates/*
- ProjectDocs/02-version-scope/*
- ProjectDocs/03-scope-approval/*
- ProjectDocs/04-requirements/*
- ProjectDocs/05-milestones/*
- ProjectDocs/09-version-documentation/*

---

## Definitions

- Mandatory template usage:
  - Every workflow-generated document must be created from its corresponding template

- Template conformance:
  - Generated documents must preserve the full required section structure of the selected template
  - Required sections must not be omitted
  - Template structure must not be modified during document creation

- Placeholder replacement:
  - All placeholders present in the selected template must be replaced in generated documents
  - No placeholder text may remain after document generation

- Type-based template selection:
  - Documentation milestones must use `ProjectDocs/templates/documentation-milestone-template.md`
  - Implementation milestones must use `ProjectDocs/templates/implementation-milestone-template.md`

---

## Invariants

- Existing validated rules must remain unchanged unless explicitly redefined
- Existing template files must not be re-aligned or redesigned in this milestone
- No implicit template selection behavior may be introduced
- All changes must preserve consistency with:
  - development-workflow.md
  - requirements
  - milestones overview

---

## Responsibilities

- Define template enforcement rules in the workflow
- Define placeholder replacement requirements
- Define conformance requirements for generated documents
- Eliminate any ambiguity regarding mandatory template usage

---

## Deliverables

- `ProjectDocs/development-workflow.md` updated with explicit template enforcement rules
- Explicit rule stating that every workflow-generated document must use its corresponding template
- Explicit rule stating that all placeholders must be replaced
- Explicit rule stating that generated documents must preserve required template structure
- Explicit rule defining type-based milestone template selection

---

## Constraints

- No runtime changes
- No redesign of template content beyond enforcement wording
- No changes outside defined workflow and documentation scope

---

## Validation Criteria

- Workflow explicitly states that template usage is mandatory
- Workflow explicitly states that placeholder leakage is forbidden
- Workflow explicitly states that required template structure must be preserved
- Workflow explicitly states type-based milestone template mapping
- No contradictory document-generation rule remains

---

## Non-Goals

- No creation of missing templates
- No re-alignment of template wording outside enforcement scope
- No implementation of external tooling for enforcement
