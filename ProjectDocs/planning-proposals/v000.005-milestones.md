# Milestones — v000.005

---

## Overview

This document defines the milestone breakdown for v0.5.

Scope is derived strictly from the approved planning proposal.

All milestones must be:

- isolated
- deterministic
- strictly scoped
- validated before proceeding

No milestone may introduce:

- implicit behavior
- cross-layer coupling
- redesign of existing components

---

## Milestones

### MS022 — scanner-multi-format-detection

- Extend scanner to recognize additional file formats:
  - PNG
  - HEIC / HEIF
  - RAW formats
  - Video formats
- Deterministic file inclusion/exclusion
- No content processing beyond detection

---

### MS023 — scanner-format-validation

- Introduce format-specific validation
- Detect unreadable or unsupported files
- Integrate with existing corrupt file classification

---

### MS024 — metadata-abstraction-layer

- Introduce a normalized metadata structure
- Decouple metadata extraction from scanner
- Maintain identical behavior for JPEG processing

---

### MS025 — metadata-extraction-extended

- Extend metadata extraction to supported formats:
  - PNG
  - HEIC
  - RAW
  - Video
- Use existing metadata only
- No inference or heuristic enrichment

---

### MS026 — metadata-normalization

- Normalize metadata across all formats:
  - timestamps
  - timestamp_source
- Ensure consistent structure and representation

---

### MS027 — pipeline-metadata-integration

- Integrate metadata layer into pipeline
- Preserve existing JPEG behavior exactly
- Ensure deterministic data flow

---

### MS028 — contextual-grouping-model

- Define contextual grouping structures
- Define grouping keys:
  - camera
  - location
  - time proximity
- Extend data model if required

---

### MS029 — contextual-grouping-rules

- Implement deterministic grouping rules
- Define fixed grouping behavior
- Ensure full reproducibility

- Must not alter:
  - duplicate grouping
  - canonical selection logic

---

### MS030 — contextual-grouping-integration

- Integrate contextual grouping into pipeline
- Execute after duplicate grouping
- Store grouping results without affecting planning

---

### MS031 — reporter-contextual-output

- Extend console output with grouping information
- Extend JSON output with grouping data
- Ensure deterministic and additive output only

---

### MS032 — end-to-end-validation

- Validate full pipeline across all formats
- Validate deterministic behavior
- Validate no regression in JPEG processing
- Validate no unintended side effects

---

## Execution Rules

- Milestones must be completed sequentially
- Each milestone must be:
  - implemented
  - validated
  - committed
- No milestone may start before the previous one is complete
- No scope deviation is allowed

---

## Success Criteria

- All milestones completed and validated
- Full alignment with planning proposal
- No regression in existing behavior
- Deterministic behavior preserved across all supported formats
